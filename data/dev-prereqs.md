## Hooks (Husky v9)

The `prepare` script in `package.json` runs on `pnpm install` and ensures
`.husky/` hooks are installed. If hooks stop firing, run `pnpm exec husky init`.

## audit_a11y.py (IS-5568 supplementary a11y check)

The supplementary IS 5568 a11y check (A11Y-08 layer-2; complements Lighthouse a11y â‰Ą0.95) is invoked via `scripts/audit_a11y_wrapper.mjs` (`pnpm qa:audit-a11y`).

**Required:**

- Python 3.9+
- pip packages: `selenium`, `axe-selenium-python` (or `requests` + `beautifulsoup4` for the offline fallback path)
- Chrome (GitHub Actions `ubuntu-latest` ships Chrome pre-installed; locally install via your package manager)

**Install locally:**

```bash
pip install selenium axe-selenium-python
```

When Python is absent, `scripts/audit_a11y_wrapper.mjs` writes a stub `data/a11y-il-results.json` with `status: "fail"` and exits 0 â€” CI surfaces the unavailability via the audit dashboard without crashing.

## Lighthouse CI (`@lhci/cli`)

The `pnpm lhci` script runs the Lighthouse 3-run-median gate against the locally-built site. **Chrome must be installed** for local runs. CI runs on `ubuntu-latest` via `.github/workflows/lighthouse.yml` (Chrome pre-installed there).

**Local invocation:**

```bash
pnpm build          # required â€” lhci runs against the production build
pnpm lhci           # boots `pnpm start`, runs 3 lhci runs per URL, asserts thresholds, persists JSON
```

**Required for the regression harness (`pnpm qa:regression`):**

- Chrome locally (the harness shells out to `pnpm lhci`)
- ~5 minutes per run (3 URLs Ă— 3 runs each + 5MB hero injection rebuild)

**Thresholds (mobile, 3-run-median):** perf â‰Ą0.90, a11y â‰Ą0.95, best-practices â‰Ą0.95, SEO 1.00.

**Output:** `data/lighthouse-results.json` (committed as empty `[]` initial state â€” regenerated by every `pnpm lhci` run, consumed by AUD-013 + AUD-034 + Quality Gate criterion 1).


A test commit that introduces a raw hex value or hard-coded partner URL must
